Dynamic Reconfiguration in Spire

C / C++ OpenSSL Docker Python

I designed and implemented a fully dynamic reconfiguration system for Spire, a fault-tolerant SCADA protection platform, enabling runtime updates to topology, cryptographic keys, and component roles without downtime across a 26-node deployment.

The Problem

Spire's original codebase relied on compile-time constants for site counts, replica numbers, and network parameters. Any topology or key change required a full recompile and redeployment — making live updates impossible and manual setup error-prone at scale. The system had no way to rotate cryptographic keys or add nodes without taking everything offline.

My Role

Sole designer and implementer of the reconfiguration system, working across the full C/C++ codebase and building the supporting deployment infrastructure from scratch.

What I Built

End-to-end reconfiguration pipeline. The system spans three components: a Config Manager that generates and signs new configurations, validating topology, roles, and key material before release; a Config Disseminator that reliably distributes packages across unreliable multi-site links; and a Config Agent running on every node that validates authenticity, stages changes, rotates keys, and coordinates local service transitions without disrupting active operations.

Hybrid cryptographic key rotation. Implemented AES-256 + RSA-OAEP encryption with key versioning so nodes can handle overlapping key sets during transitions and securely discard deprecated material — without any window where the system is unprotected.

Custom reliable multicast protocol. Built fragmentation, acknowledgment tracking, and retransmission logic from scratch to guarantee complete configuration delivery across all nodes before any transition begins. This was necessary because standard reliable transport assumptions don't hold across Spire's multi-site links.

26-node reproducible demo environment. To validate under realistic conditions, I built a custom AlmaLinux-based Docker image that compiles the full Spire stack from source, automated all dependency and key generation steps, and designed a Docker Compose topology of 26 containers on a deterministic bridged subnet. The result is a one-command environment for demonstrating live topology changes, key rotation, and coordinated transitions with zero downtime.
